# Contributing to Code Search

We love contributions! To keep things organized and legally sound for our open-core model, please follow these guidelines.

## How to Contribute

1. **Check for Issues:** Before starting work, please check the [Issue Tracker](https://github.com/techquestsdev/code-search/issues) to see if the feature or bug is already being discussed.
2. **Sign the ICLA:** By contributing to this project, you agree to the terms of the [Individual Contributor License Agreement (ICLA)](./ICLA.md). This allows us to keep the core free while also sustaining the project via enterprise features.
3. **Fork and Branch:** Fork the repo and create a branch for your changes.
4. **Follow the Style:** Please ensure your code follows the existing style and conventions of the project (Go, TypeScript, etc.).
5. **Tests:** Ensure your changes are covered by tests.
6. **Submit a PR:** Provide a clear description of your changes and why they are needed.

## Legal Note

By submitting a Pull Request, you represent that your contribution is your original work and that you have the right to grant us the licenses described in the [ICLA](./ICLA.md) and the [LICENSE](./LICENSE).
